# Editorial Checklist

Use this checklist before shipping curriculum edits.

- Audience: does the lesson clearly help a business individual contributor do real work?
- Accuracy: are named tool claims, numbers, or workflow assertions still true as of the lesson's `lastReviewed` date?
- Specificity: does the lesson use concrete work outputs instead of generic AI talk?
- Learning quality: does it include a clear outcome, example, artifact, and under-10-minute action?
- Prompt example: does the lesson include a usable example prompt that matches the actual workflow being taught?
- Sequence: does `sequenceRationale` explain why this lesson sits here in the module?
- Verification: does it say what must be checked before using the output?
- Role relevance: do the manager, writer, analyst, and operator variants add useful perspective?
- Resources: if links are present, are they official or clearly justified and still worth the learner's attention?
- Maintainability: can a future editor update the lesson without deciphering raw HTML?
